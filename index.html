Introduction & Characteristics of Computers

A computer is an electronic machine that can calculate, process data, and control tasks. It automatically processes input parameters under systemic programmatic instructions to render meaningful results.

The Information Processing Cycle:

• Input: Giving data to a computer.

• Processing: Following instructions to manipulate variables.

• Output: Showing the result on a screen.

• Storage: Saving data for future use.

Core Performance Indicators

Speed: Processes millions of instructions per second (MIPS) instantly.
Accuracy: Works perfectly without mistakes. Output failures point directly to user error or faulty data parameters via GIGO (Garbage In, Garbage Out).
Diligence: Can work continuously over long durations without exhaustion, fatigue, or loss of focus.
Versatility: Capable of managing multi-tasking operations and multiple completely different types of processing tasks sequentially.
  • Automation: Can perform complex sequential tasks automatically once set up initially.

Evolutionary History & Pioneers

The progression of computational technology from mechanical counting logic to modern computer networks:

  • Abacus: The earliest historical counting device.
  • Pascaline: The first automatic mechanical calculator, restricted strictly to addition and subtraction ($+$ and $-$).
  • Stepped Reckoner: An improved mechanical calculator that could multiply, divide, and find square roots.
  • Punch Cards: Used in early mechanical computers to feed input data parameters.
  • Charles Babbage: Recognized historically as the "Father of the Computer".
  • Ada Lovelace: Acclaimed as the world's very first computer programmer.
  • ENIAC: The first automatic, general-purpose, electronic digital computer.
  • EDVAC: Introduced the groundbreaking stored-program concept and shifted computing hardware to use the Binary System ($0$ and $1$).
  • UNIVAC: The first digital computer built explicitly for commercial corporate use.

Classification of Computers

Computing frameworks are classified into four major domains based on processing scale, user capabilities, and hardware capacity:

  • Microcomputers: Small, low-cost computer layouts built explicitly for a single user (e.g., Laptops, Smartphones, Desktop personal computers).
  • Minicomputers: Medium-sized computing systems optimized to handle multiple users simultaneously (e.g., Apple Mac Mini or departmental server platforms).
  • Mainframe Computers: Large, highly powerful systems engineered to manage massive operations and thousands of concurrent users safely (e.g., Core Banking Systems).
  • Supercomputers: The fastest, most expensive, and most powerful systems in the world. They utilize massive parallel processing power to manage complex computational assignments (e.g., Weather Forecasting setups).

Computer Architecture & System Buses

Modern computers operate under the standard Von Neumann Architecture, which describes a configuration where instructions and data variables share a single memory space and are executed in a sequential pipeline.

Central Processing Unit (CPU)

  • Control Unit (CU): Acts as the manager of the computer. It coordinates system tasks and handles instruction Fetch and Decode logic steps.
  • Arithmetic Logic Unit (ALU): Performs all computing calculations.
    • Arithmetic: $+$, $-$, $\times$, $\div$
    • Logical: $AND$, $OR$, $NOT$
  • Registers: Tiny, extremely high-speed local memory cells inside the CPU chip used to hold instantaneous data temporary variables.

The System Bus Matrix

  • Address Bus (Unidirectional): Carries target memory physical location data addresses directly from the CPU out to system components.
  • Data Bus (Bidirectional): The main communications highway that shifts actual data bits back and forth across core modules.
  • Control Bus (Bidirectional): Transmits system control instructions and synchronization timing clock signals to maintain operations.
The Instruction Execution Cycle:

1. Fetch: The CPU takes the next instruction from the main memory module.

2. Decode: The Control Unit translates the instruction to determine what action is required.

3. Execute: The ALU or relevant component performs the required task.

Hardware, Storage Subsystems & Ports

I/O Peripherals

Input Devices: Provide external parameters to the computer (e.g., Keyboards for text input, Mice for making selections, and Scanners for converting paper documents into digital archives).

Output Devices: Display the final processed outcomes (e.g., Monitors to view digital content, Printers to generate hard copies, and Speakers to output audio sounds).

The Memory Hierarchy Matrix

  • Cache Memory: Extremely fast, low-capacity storage situated close to or inside the CPU chip to cache frequently used data.
  • Primary Storage (RAM): Main memory. It holds programs currently running and is completely volatile, meaning all data is erased instantly if the power source is cut off.
  • Read Only Memory (ROM): Permanent, non-volatile memory space that safely preserves the critical initial startup instructions (BIOS/firmware) needed to boot the hardware.

RAM Variants

  • SRAM (Static RAM): Extremely fast, expensive memory that requires no refreshing cycles; used primarily for CPU Cache.
  • DRAM (Dynamic RAM): Slower and cheaper than SRAM; needs constant electrical refreshing and serves as standard main memory.
  • SDRAM (Synchronous DRAM): Faster than traditional DRAM; aligns with the system clock and is used in modern main memory.

ROM Variants

  • PROM (Programmable ROM): A blank chip manufactured to be programmed by a user exactly once.
  • EPROM (Erasable PROM): Can be completely wiped using strong Ultraviolet (UV) Light and rewritten.
  • EEPROM (Electrically Erasable PROM): Can be cleared and rewritten electrically byte-by-byte.

Secondary Persistent Storage

Used to store user files permanently. Unlike RAM, it is non-volatile.

  • HDD (Hard Disk Drive): Traditional storage using rotating magnetic platters. It is cheap and offers high capacity, but operates slowly and is vulnerable to drop damage.
  • SSD (Solid State Drive): Modern storage using flash memory. It contains zero moving parts, runs exceptionally fast, and is highly durable.

Flash & Optical Format Details

  • USB Flash Drives: Portable flash memory devices ranging from $1\text{ GB}$ up to $256\text{ GB}$.
  • Memory Cards (SD / microSD): High-density micro-storage units from $2\text{ GB}$ up to $1\text{ TB}$, used in cameras and mobile phones.
  • Compact Disc (CD): Stores up to $700\text{ MB}$ of data and relies on an Infrared Laser ($780\text{nm}$).
  • Digital Versatile Disc (DVD): Stores $4.7\text{ GB}$ on a standard single-layer disc and relies on a Red Laser ($650\text{nm}$).
  • Blu-ray Disc: Stores up to $25\text{ GB}$ on a single layer and relies on a short-wavelength Blue-Violet Laser ($405\text{nm}$).

Cloud Storage Frameworks

Off-site file storage hosted on remote networks, offering easy sharing and anytime access without requiring local hardware upkeep.

  • Public Cloud: Shared platforms accessible over the public internet (e.g., Google Drive, Microsoft OneDrive).
  • Private Cloud: Dedicated server networks setup exclusively for a single secure organization.
  • Hybrid Cloud: Combined systems that blend private infrastructure control with public cloud scaling options.

Number Systems & Character Encoding

Radix Bases Matrix

  • Decimal System (Base 10): Uses digits $0-9$.
  • Binary System (Base 2): Uses bits $0$ and $1$.
  • Octal System (Base 8): Uses digits $0-7$.
  • Hexadecimal System (Base 16): Uses digits $0-9$ and letters $A-F$ ($A=10, B=11, C=12, D=13, E=14, F=15$).

Conversion Work Examples

Decimal to Binary Conversion ($25_{10}$):

Divide by 2 repeatedly and read the remainders from bottom to top:

$25 \div 2 = 12 \text{ with Remainder } \mathbf{1}$

$12 \div 2 = 6 \text{ with Remainder } \mathbf{0}$

$6 \div 2 = 3 \text{ with Remainder } \mathbf{0}$

$3 \div 2 = 1 \text{ with Remainder } \mathbf{1}$

$1 \div 2 = 0 \text{ with Remainder } \mathbf{1}$

Resulting Binary Sequence: $\mathbf{11001_2}$

Binary to Decimal Conversion ($11001_2$):

Multiply each bit position by its corresponding power of 2:

$(1 \times 2^4) + (1 \times 2^3) + (0 \times 2^2) + (0 \times 2^1) + (1 \times 2^0)$

$= 16 + 8 + 0 + 0 + 1 = \mathbf{25_{10}}$

Binary Shortcut Mechanics

  • Binary to Octal: Group binary bits into clusters of three starting from the right-hand side, then write out the decimal value for each cluster (e.g., $101110_2 \rightarrow \underbrace{101}_{5} / \underbrace{110}_{6} \rightarrow \mathbf{56_8}$).
  • Binary to Hexadecimal: Group binary bits into clusters of four starting from the right-hand side, using the $8-4-2-1$ positional layout to find the character (e.g., $10111100_2 \rightarrow \underbrace{1011}_{11 \rightarrow B} / \underbrace{1100}_{12 \rightarrow C} \rightarrow \mathbf{BC_{16}}$).

Character Encoding Systems

  • ASCII: A $7\text{-bit}$ character encoding scheme that maps exactly $128$ characters, covering standard English letters, numbers, and basic punctuation marks.
  • EBCDIC: An $8\text{-bit}$ character encoding standard created by IBM that supports up to $256$ characters; used primarily within legacy corporate mainframe environments.
  • Unicode: A universal character mapping system engineered to support all international written languages and modern emojis. It maps over $149,000$ distinct characters using variable lengths (such as UTF-8 and UTF-16) and identifies each character by an explicit hexadecimal value called a Code Point.

Algorithms, Flowcharts & Control Structures

Logic Definitions

  • Algorithm: A clear, step-by-step procedure designed to solve a specific problem.
  • Pseudocode: A detailed yet readable plain-text description of an algorithm that mimics programming logic without strict language syntax.

Flowchart Visual Standards

Flowcharts map the logical execution path of an algorithm using standard geometric shapes:

  • Oval: Indicates the Start or End point of a program.
  • Parallelogram: Represents Input or Output operations.
  • Rectangle: Denotes arithmetic processing calculations or internal data manipulation.
  • Diamond: Specifies a Decision point or conditional evaluation step.
  • Arrows: Indicate the explicit direction of the process flow.

Structural Control Patterns

  1. Sequence: Instructions execute sequentially one after another, moving from top to bottom.
  2. Selection: Checks a specific condition and branches along different paths based on the result.
  3. Repetition (Looping): Repeatedly executes a block of code as long as a specified condition remains true.
